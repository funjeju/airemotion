# 06 · BUILD PHASES — 단계별 구현 로드맵

> 각 Phase는 **검증 기준**을 통과해야 다음으로 넘어간다. 한 번에 전부 만들지 않는다.

## Phase 0 — 프로젝트 셋업
- Next.js(App Router, TS) + Tailwind + `next-themes` + `next-intl` 초기화.
- 디자인 토큰(05 문서) → Tailwind 설정에 컬러/폰트/라운드 반영.
- 폰트 로드: Space Grotesk / Inter / Geist Mono.
- `messages/ko.json`, `messages/en.json` 골격. `/[locale]` 라우팅.
- 상단바: 로고 + 언어 셀렉터 + 테마 토글(동작).
- **검증:** 한/영 전환되고, 다크/라이트 토글이 모든 기본 화면에서 정상 동작한다.

## Phase 1 — 인증 + 대시보드
- Firebase 연동(02·03 문서). **Google 로그인**, 보호 라우트.
- 최초 로그인 시 `users/{uid}` 생성(locale·theme 저장).
- 대시보드: 내 프로젝트 목록 + "새 프로젝트".
- **검증:** 구글로 로그인/로그아웃되고, 본인 프로젝트만 보이며, 미인증은 login으로 막힌다.

## Phase 2 — 업로드 + 자동 배열 + 미리보기 (서버 렌더 없음, MVP 핵심)
- 다중 업로드 → Firebase Storage, 메타데이터 → Firestore `clips`.
- 필름스트립 타임라인(시그니처): 드래그 순서 정렬.
- 사진 기본 4초, `zoomIn/zoomOut/pan` 번갈아 자동 배정 + 크로스페이드.
- 클립별 **자막 입력**(전역 기본값 + 개별 오버라이드).
- 브라우저 **재생 미리보기**(켄 번스 + 크로스페이드 + 자막).
- **검증:** 사진 올리고 순서·자막만 정하면, 버튼 없이도 자동 효과가 적용된 미리보기가 재생된다.

## Phase 3 — Remotion + 영상 클립 + 음악 + MP4 출력
- Remotion 컴포지션으로 전환(`TransitionSeries`, `@remotion/transitions`).
- 영상 클립 지원 + 배경음악 트랙. FFmpeg로 길이 정합(영상에 음악 맞춤).
- 렌더 트리거(API Route → 렌더 워커). **Vercel 함수에서 직접 렌더 금지.**
- 우선 로컬/단일 워커 렌더로 MP4 출력 검증.
- **검증:** 사진·영상·음악·자막이 합쳐진 MP4가 정상 출력된다.

## Phase 4 — 자동 자막(Whisper) + 검수
- 음성 소스 → Whisper(OpenAI API 또는 워커 내) → 자막 초안.
- **렌더 전** 검수 UI: 텍스트·타이밍 수정 → 확정 → 렌더.
- 한국어 다국어 모델, 오인식 대비 검수 필수.
- **검증:** MP3/영상 음성에서 자막 초안이 생성되고, 수정·확정 후 그 자막이 최종 영상에 반영된다.

## Phase 5 — 제품화 + 클라우드 병렬 렌더
- 업로드·정렬·검수 UI 다듬기, 프롬프트 톤 매핑(calm/lively).
- 자막 세부 옵션(색·배경·크기·위치) 인스펙터 완성.
- **Remotion Lambda(AWS)** 병렬 렌더로 전환. 렌더 전 `estimatePrice`로 비용 표시.
- 대용량 에셋 대역폭 절감(예: Cloudflare R2) 검토.
- **검증:** 여러 사용자가 동시에 렌더해도 빠르게 처리되고, 예상 비용이 표시된다.

## 공통 품질 기준 (모든 Phase)
- 라이트/다크 양쪽 검증, 한/영 양쪽 검증.
- 키보드 포커스·반응형·`prefers-reduced-motion` 충족.
- 시크릿은 서버사이드/환경변수만. `.env*`는 깃 제외.

## 구축 중 사용자에게 확인할 것 (추측 금지)
- Remotion 라이선스 적용 여부(회사 규모 시 유료).
- 클라우드 렌더 비용/AWS·GCP 요금(변동) — 짧은 테스트 렌더로 실측 권장.
- 한국어 STT 정확도 — 필요 시 한국어 특화 STT 비교.
